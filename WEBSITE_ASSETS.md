# Website Assets Inventory (Sessiono S2M)

## Goal

Define the complete asset set needed for a front-to-back one-iteration website launch.

## 1) Brand assets

- Primary logo (light/dark variants)
- Icon mark
- Favicon set
- OG/social share image templates
- Typography pairings and token mapping

## 2) Marketing and storytelling assets

- Homepage hero art (creator + session workflow)
- Product explainer visuals (brief -> generate -> hire -> deliver)
- Testimonial and case-study card templates
- Pricing/plan comparison graphics
- Trust and safety callout assets

## 3) Product UI assets

- Navigation and dashboard icon set
- Empty/loading state illustrations
- Project status badges (briefing, in-progress, revision, complete)
- Marketplace profile card imagery patterns
- Milestone/payment flow UI states

## 4) Operations and support assets

- Onboarding checklist components
- Help center visual snippets
- Email template graphics for project/milestone events
- Admin moderation status indicators

## 5) Content and SEO assets

- Metadata copy blocks for landing and key feature pages
- Structured FAQ snippets
- Feature-specific screenshot placeholders
- Creator/musician segment landing-page copy variants

## Asset quality standards

- All assets versioned and named predictably (`asset-area-purpose-vX`)
- Accessibility review for color contrast and legibility
- Rights ownership recorded for all visual and audio promotional assets
